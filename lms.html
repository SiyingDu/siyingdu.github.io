DoubleVerify · Overcoming Platform Limitations

Context

DoubleVerify's new LMS launched with a fundamental mismatch: the platform couldn't support the org structure, the learner flow was broken, and admins needed to write code to customise anything.

Three layers of failure

2-level content limit couldn't support nested department and team structures

Navigation broke on multi-topic plans — assessments surfaced mid-program, learners dropped off

Admins needed HTML skills to customise pages — creating an engineering dependency for every update

Three architectural decisions

Decision 1

Content structure

Used multi-level communities + homepage filters to create 4+ level hierarchy — working around the platform's 2-level limit without a rebuild.

Decision 2

API-driven front end

Leveraged LMS API to control rendering order, gate assessments until completion, and sequence topics correctly — decoupling experience from platform constraints.

Decision 3

Excel Macro admin tool

Built a one-click tool generating branded, interactive navigation pages from a spreadsheet — no HTML required. Admin self-serve from days to under 10 minutes.
